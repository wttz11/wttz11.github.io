模块化

2023-11-17

CommonJS

关键词：

  • 社区标准

  • 使用函数实现

  • 仅node环境支持

  • 动态依赖（需要代码运行后才能确定依赖），动态依赖是同步执行的

  • 导出的是值的拷贝，模块中内容在导出后发生变化不会影响模块外

  • 模块可以被多次引用、加载。在第一次被加载时，会被缓存 ，之后都从缓存中直接读取结果。

初始时，module.exports与exports指向同一块地址，模块默认优先导出module.exports指向的地址，所以直接修改exports的值导致其与module.exports指向不同地址之后，导出会出现问题。优先是由module.exports进行导出！！！

原理：

// require函数的伪代码
function require(path){

    //Module类似一个全局变量，管理所有模块
    const  cachedModule = Module._cache[id]
   
   /* 如果已经加载了那么直接取走缓存的 exports 对象  */
    if(cachedModule){
        return cachedModule.exports
    }

    function _run(exports, require, module, __filename, __dirname){
    // 模块代码会放到这里
    }
  
    var module = {
        exports: {}
    }

    //执行模块代码
    _run.call(
        module.exports, 
        module.exports, 
        require, 
        module, 
        模块路径,
        模块所在目录
    );
  

    Module._cache[id] = module

    return module.exports;
}

ES Module

关键词：

  • 官方标准

  • 使用新语法实现

  • 所有环境均支持

  • 同时支持静态依赖和动态依赖

    静态依赖：在代码运行前就要确定依赖关系

  • 动态依赖是异步的(promise)

  • 符号绑定,ES模块输出的是值的引用

关于符号绑定：

// module a.js
export var a = 1;
export function changeA(){
  a = 2;
}

// index.js
// 导入位置的符号和导出的符号并非赋值，它们完全是一个东西
import {a, changeA} from './a.js';
console.log(a); // 1
changeA();
console.log(a); // 2
